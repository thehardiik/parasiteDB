# ParasiteDB

A Node.js library that uses **Google Sheets as a cloud database**. Provides schema definition, validation, and full CRUD operations backed by the Google Sheets API.

## Architecture

```
app.js          Express REST API server (entry point)
DB.js           Google Sheets connection (singleton via static class)
Schema.js       Core logic: schema definition, validation, CRUD operations
hashing.js      Generates numeric hash IDs from primary key values
```

## How It Works

1. `DB.connectDB(email, key, sheetID)` authenticates with a Google service account and opens the spreadsheet.
2. A `Schema` is defined with attribute names, data types, required flags, and one primary key.
3. Each schema creates its own **sheet tab** in the Google Spreadsheet. Row 1 is the header with constraint annotations (`P`/`NP` for primary key, `R`/`NR` for required).
4. CRUD operations are implemented by **writing Google Sheets formulas** (FILTER, ARRAYFORMULA, MATCH, ROW) into a dedicated "Query" sheet, then reading back the computed values.
5. The `_id` field is auto-generated by hashing the primary key value.

## Schema Definition

```js
const schema = new Schema("sheetName", {
  caption: { dataType: "String", required: true, primaryKey: true },
  id:      { dataType: "Integer", required: true },
  likes:   { dataType: "Integer", required: false },
});
await schema.initialiseInSheets("create"); // "create" | "connect"
```

## Key Implementation Details

- **Query sheet**: Used as scratch space for formulas. Results land in A1:Z1.
- **Column mapping**: Attributes map to A1 notation (A, B, C…). `_id` is always column A.
- **Type handling in formulas**: String values are quoted in filter expressions; Integers are not.
- **Duplicate detection**: `createData` uses a FILTER formula to check for existing primary keys before inserting.
- **Row targeting**: `updateOne`/`deleteOne` use `FILTER + ROW` to find the row index, then address cells directly.
